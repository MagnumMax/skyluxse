6. РУКОВОДСТВО ПО РАЗВЕРТЫВАНИЮ (DEPLOYMENT GUIDE)

6.1. Требования к окружению

  • Node.js v20+
  • npm / pnpm
  • PostgreSQL 15+

6.2. Процесс развертывания (Deployment Pipeline)

Проект настроен на автоматический деплой через Vercel при пуше в ветку main.

Ручная сборка и запуск:

# Установка зависимостей npm install # Применение миграций БД npx supabase migration up # Сборка проекта npm run build # Запуск npm run start

6.3. Резервное копирование (Backup)

Настроено автоматическое ежедневное резервное копирование базы данных средствами Supabase (Point-in-Time Recovery). Период хранения: 7 дней (на тарифе Pro).

6.4. Мониторинг

  • Ошибки приложения: Vercel Logs / Sentry (если подключен).
  • Статус интеграций: Таблица integrations_outbox в БД (проверка поля status).

7. ИНСТРУКЦИИ ПОЛЬЗОВАТЕЛЯ (USER MANUALS)

7.1. Руководство Администратора (Admin Guide)

Доступ: /exec/dashboard

  • Управление бронированиями: Перейдите в раздел "Bookings". Канбан-доска отображает текущий статус. Для смены статуса перетащите карточку.
  • Календарь автопарка: Раздел "Fleet Calendar" показывает доступность машин. Конфликты подсвечиваются красным.
  • Синхронизация с Zoho: Происходит автоматически. В случае ошибки, статус виден в "Integrations Dashboard".

7.2. Руководство Водителя (Driver Guide)

Доступ: /driver/tasks (оптимизировано для мобильных устройств)

  1. Войдите в систему под своим аккаунтом.
  2. На главном экране выберите активную задачу (Task).
  3. Заполните чек-лист (Checklist).
  4. Загрузите фото (до/после аренды) и укажите пробег/топливо.
  5. Нажмите "Complete Task". Данные синхронизируются с офисом автоматически.

Примечание: Приложение работает без интернета. Данные отправятся, как только связь появится.
