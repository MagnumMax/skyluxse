Я проанализировал кодовую базу и вижу, что в проекте используется библиотека компонентов Shadcn UI (`components/ui/button.tsx`), а формы реализованы несколькими способами (React Hook Form, Server Actions, ручные формы).

Предлагаю системный подход, чтобы не дублировать код спиннера в каждой форме:

### 1. Модернизация базового компонента Button
В файле `components/ui/button.tsx` добавлю проп `isLoading`.
- Если `isLoading={true}`, кнопка автоматически:
  - Блокируется (`disabled`).
  - Показывает спиннер (`Loader2` из `lucide-react`).
  - Сохраняет свои размеры и стили.

Это позволит использовать простой синтаксис везде:
```tsx
<Button isLoading={isSubmitting}>Сохранить</Button>
```

### 2. Внедрение в существующие формы
Я нашел ключевые формы, которые обновлю сразу после модернизации кнопки:

1.  **`components/driver-task-form.tsx`** (Форма задач водителя)
    *   *Сейчас:* Использует ручную верстку `<Loader2 />` и проверку `isSubmitting`.
    *   *Будет:* Заменю на `<Button isLoading={isSubmitting} ...>`.

2.  **`components/fleet/vehicle-form.tsx`** (Форма автомобиля)
    *   *Сейчас:* Использует React Hook Form.
    *   *Будет:* Добавлю `isLoading={form.formState.isSubmitting}`.

3.  **`components/sales-service-form.tsx`** (Форма оценки сервиса)
    *   *Сейчас:* Server Action с `useFormStatus`.
    *   *Будет:* Обновлю локальный компонент `SubmitButton` для использования нового пропа.

4.  **`components/zoho/create-sales-order-button.tsx`** (Кнопка создания заказа)
    *   *Сейчас:* Меняет текст на "Creating...".
    *   *Будет:* Использовать спиннер через `isLoading={isPending}`.

### 3. Результат
Вы получите единообразное поведение (UI/UX) во всем приложении: при нажатии кнопка блокируется и показывает индикатор загрузки.
