1. Проверьте наличие отладочных (debug) или тестовых файлов, созданных в рамках текущего диалога, которые не были закоммичены. Убедитесь, что эти файлы нигде в коде не используются. Если они не являются частью логики системы, удалите их.

2. Проанализируй незакоммиченные файлы, применяя метод Sequential Thinking, и составь детализированный список выполненных задач. Для каждой задачи укажи:
- Ключевые параметры реализации
- Конкретные требования, которые были выполнены
- Связь с контекстом диалога
- Соответствующие изменения в файлах
Список должен быть структурированным и содержать только подтверждённые выполненные задачи.

3. Для каждой задачи из составленного списка:
- Получи лучшие практики из контекста MCP (context7),
- Проанализируй код на соответствие этим практикам и выяви конкретные возможности для улучшения,
- Составь детализированный итоговый список рекомендуемых улучшений с указанием конкретных участков кода и предлагаемых изменений.
   
4. Запусти валидацию кода:
   - Выполни компиляцию через TypeScript
   - Проведи линтинг через ESLint
   - Устрани все выявленные ошибки
